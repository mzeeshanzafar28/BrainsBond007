Admin Side

Login/Sign Up

The admin of BrainsBond007 can log in or sign up to access the system's management dashboard. This section will feature secure login credentials (username, email, password), and potentially two-factor authentication to enhance security. The admin panel will be the control center for managing employee details and monitoring performance.

Upload Employee Details

Admins can upload employee details. Each entry will include:

  • Name
  • Age
  • Email
  • Images
  • CNIC/ID
  • Working Hrs
  • Allow Remote (y/n)
  • Allowed Remote Locations
The system allows editing and deleting employees, giving the admin full control over the employee database.

Upload Office/Work Locations

Admins can upload specific office locations. This is particularly useful for tracking remote employees and comparing their actual location to the allowed office or remote work locations using geofencing technology. Google Maps API will assist in defining and managing these locations.

Admin Dashboard

The admin dashboard will display the following key sections:

  • Employee Stats: Overall performance, attendance, and activities of employees.
  • Charts/Graphs: Visual statistics based on attendance, working hours, and remote job activity.
  • Settings: Manage security preferences, employee access levels, and general system settings.
The dashboard will allow the admin to view screenshots and images of employees working, as well as live webcam feeds.

Live Event Monitoring

Admins will be able to monitor employees live through the webcam. Screenshots and random images will be captured during work hours for verification purposes. This will require using a streaming API, potentially something like WebRTC, since face-api.js is primarily designed for static image processing.

Create Dynamic EXE for Each Admin/User

Admins can create a dynamic executable (EXE) file for each admin or user. This EXE will include custom configurations, allowing for individualized use of the system.

Allow/Disallow Remote/Onsite for Each Employee

The admin can now set whether an employee is allowed to work remotely or must be present onsite. This option provides more control over employee working conditions and permissions.

Employee Side

Location Verification

For remote employees, the system checks their current location using the Google Maps API. If they are working remotely, the system compares their location with predefined office or job site locations (up to 3 allowed locations). If the employee is not within the permitted range, a notification is sent to the admin for further action.

Camera Access & Face Match

The system will access the employee's camera and take a picture upon logging in. The face recognition algorithm (using face-api.js) will compare the captured image to the one stored in the system. If the match score is less than 70%, the system will request an ID card scan for further verification.

Session Monitoring

Once authenticated, the system will begin the session. During the session, random screenshots will be captured along with random webcam images. This helps ensure that the employee is active and working within their designated working hours. The system will automatically capture 5 images at random intervals throughout the session.

Live Screen Casting

The system now enables live screen casting. Admins can view an employee's screen in real-time to monitor activities and ensure compliance with the assigned tasks.

Seize System

Admins have the ability to seize control of an employee's system remotely. This feature can be used in cases of suspicious behavior or unauthorized activities.

Screen Away Monitoring

In addition to random screenshots, the system will monitor if an employee moves away from their screen. Alerts will be triggered if the system detects prolonged inactivity.

End Time Attendance

Prompt the employee for attendance at the end of working hours.

Devised Technologies

Admin Side Technologies

The following technologies are used for developing the admin panel and backend of the BrainsBond007 project:

  • Laravel - Backend framework
  • Vue - Frontend framework for interactive UI
  • Tailwind - CSS framework for design and layout

APIs

The project integrates with external APIs to enhance functionality:

  • Google Maps API - For location verification and geofencing

Client-Side EXE Technologies

The following technologies are used in the client-side executable (EXE) development for the BrainsBond007 system:

  • Tensorflow - Machine learning library for face recognition
  • Tkinter - Python GUI toolkit for creating desktop applications
  • py2exe - Tool for converting Python scripts into Windows executables
  • ScreenCapture - For capturing screenshots and live screencast of the employee's desktop
  • CTYPES - For seizing the employee system in case of Misuse/Prohibited Action
